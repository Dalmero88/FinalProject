## О проекте

Проект посвящён сравнению трёх подходов к классификации музыкальных жанров по изображениям мел-спектрограмм:

| # | Подход | Описание |
|---|--------|----------|
| 1 | **MLP (baseline)** | Простая полносвязная сеть из урока «Практика: MLP для классификации музыкальных жанров» |
| 2 | **CNN** | Собственная свёрточная сеть: 32 → 64 → 128 каналов |
| 3 | **EfficientNet B0** | Предобученная модель с Transfer Learning (ImageNet) |

**Фреймворк:** PyTorch  
**Среда обучения:** Google Colab (GPU Tesla T4)  
**Метрика:** Accuracy (точность классификации)

## Датасет

**GTZAN** — классический датасет для задачи классификации музыкальных жанров («MNIST для звука»).

- **1000 аудиотреков** по 30 секунд каждый
- **10 жанров:** blues, classical, country, disco, hiphop, jazz, metal, pop, reggae, rock
- **999 спектрограмм** (один файл отсутствует — штатная ситуация для реальных данных)
- **CSV с 57 признаками:** MFCC (×20), chroma, RMS, spectral centroid, tempo и др.

**Разбивка данных:** Train 80% / Val 10% / Test 10%

```
Датасет: andradaolteanu/gtzan-dataset-music-genre-classification (Kaggle)
```

## Архитектуры моделей

### Задание 1 — Свёрточная нейронная сеть (CNN)

```
Вход: 128×128×3
  │
  ├─ Conv2d(3→32, 3×3) + BatchNorm2d + ReLU + MaxPool(2×2)  →  64×64×32
  ├─ Conv2d(32→64, 3×3) + BatchNorm2d + ReLU + MaxPool(2×2) →  32×32×64
  ├─ Conv2d(64→128, 3×3) + BatchNorm2d + ReLU + MaxPool(2×2)→  16×16×128
  │
  ├─ Flatten  →  32768
  ├─ Linear(32768 → 256) + ReLU
  ├─ Dropout(p=0.3)
  └─ Linear(256 → 10)
```

**Параметры обучения:**
- Оптимизатор: `AdamW` (lr=1e-3, weight_decay=1e-4)
- Планировщик: `ReduceLROnPlateau` (factor=0.5, patience=5)
- Эпох: **50**
- Batch size: 32

### Задание 2 — Transfer Learning (EfficientNet B0)

```
Предобученный EfficientNet B0 (ImageNet)
  │
  ├─ [ЗАМОРОЖЕНО] features (MBConv блоки) — 4,007,548 параметров
  │
  └─ [ОБУЧАЕМЫЙ] classifier:
       Dropout(0.4) → Linear(1280→256) → ReLU → Dropout(0.3) → Linear(256→10)
```

**Аугментация (только train):**
- `RandomHorizontalFlip(p=0.5)`
- `RandomRotation(degrees=10)`
- `ColorJitter(brightness=0.2, contrast=0.2)`

**Параметры обучения:**
- Вход: 224×224 (стандарт EfficientNet)
- Оптимизатор: `Adam` (lr=1e-3)
- Планировщик: `ReduceLROnPlateau` (factor=0.5, patience=4)
- Эпох: **30**
- Batch size: 32

## Выводы

**EfficientNet B0 с Transfer Learning показала наилучшие результаты** среди всех трёх подходов.

Это объясняется несколькими факторами:

1. **Предобученные признаки (Transfer Learning).** Сеть, обученная на ImageNet, уже «умеет» выделять универсальные визуальные паттерны — края, текстуры, градиенты. Для классификации спектрограмм ей достаточно было адаптировать лишь финальный классификатор, а не учиться «видеть» с нуля.

2. **Compound Scaling (архитектура EfficientNet).** В отличие от наивного увеличения только глубины или ширины, EfficientNet балансирует все три измерения одновременно, что обеспечивает лучшее соотношение точности и числа параметров.

3. **Аугментация данных.** На небольшом датасете (999 образцов) аугментация критически важна — она искусственно расширяет обучающую выборку и снижает переобучение.

4. **CNN vs MLP.** Свёрточная сеть значительно превзошла MLP благодаря пространственной локальности: свёрточные фильтры улавливают локальные паттерны (ритмические структуры, текстуры), которые MLP полностью игнорирует, рассматривая каждый пиксель изображения независимо. MLP с 6 млн параметров уступает CNN с 1.7 млн — это наглядный пример того, что количество параметров само по себе не определяет качество модели.

5. **Парадокс параметров (MLP baseline).** 6.4 млн параметров против 330 тыс. обучаемых у EfficientNet — и при этом MLP даёт втрое меньшую точность. Правильное представление данных и архитектурное соответствие задаче важнее размера модели.
